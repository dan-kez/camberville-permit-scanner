What it does

Permit Scanner monitors building permit activity near Davis, Porter, Central, Inman, and Union squares in Cambridge and Somerville, MA, looking for patterns that suggest a property is being prepared for sale.

It combines public permit data, property assessment records, and an LLM evaluation to surface the most likely sale-prep candidates — high-value renovation clusters at non-owner-occupied or recently purchased properties.

Pipeline

  1. Fetch — Pull permits and property records from Socrata and ArcGIS APIs; cache raw JSON locally in .cache/
  2. Filter — Keep only permits within the configured radius of target squares; apply a minimum cost threshold for Cambridge alteration permits
  3. Score — Assign a significance score to each permit based on keywords, cost, completion status, and owner-occupancy flags
  4. Enrich — Join each address to the property assessment database to add year built, sale history, bedrooms, assessed value, etc.
  5. LLM assess — Send each address's permit + property summary to Claude Sonnet for a likelihood assessment (high / medium / low)
  6. Build site — Compile all assessments into docs/data.json and serve the static HTML dashboard

Scoring

Each permit gets a numeric score; the sum drives the main table sort. Higher scores surface addresses with more renovation signals.

FactorPoints
Significant keyword match (gut, renovate, addition, etc.)+3
Non-owner-occupied+2
Cost ≥ $100k+2
Cost ≥ $25k (moderate)+1
Permit completed/closed+1
All permits at address completed+1
Minor keyword match (solar, roof, windows…)+1 (reduced)

LLM assessment

Each address summary is sent to Claude Sonnet via the claude CLI. The prompt instructs the model to assess sale-prep likelihood based on the permit descriptions, costs, owner-occupancy, and property history.

Key signals the LLM weighs:

  • Simultaneous kitchen + bathroom renovations → high likelihood
  • Non-owner-occupied property with significant work → high likelihood
  • Maintenance-only work (roof, HVAC, solar) by long-term owner → low likelihood

Addresses are processed in parallel (up to 10 at once) to keep run time reasonable.
